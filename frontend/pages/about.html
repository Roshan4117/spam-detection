How SpamShield Works

// dual-model ensemble · url threat analysis · real-time classification

01 — The Problem

Spam and phishing attacks cost individuals and organizations billions annually. Traditional rule-based filters miss novel attacks. SpamShield uses two complementary machine learning models that cover each other's weaknesses through ensemble voting.

02 — The ML Models

⚡ Support Vector Machine (SVM)

A linear kernel SVM trained on TF-IDF bigram features. SVMs find the optimal hyperplane that separates spam from ham with maximum margin — making them robust to overfitting and excellent at generalizing to unseen spam patterns.

🧠 Naïve Bayes (MultinomialNB)

A probabilistic classifier using Bayes' theorem with the naïve independence assumption. Fast, interpretable, and particularly strong at catching keyword-heavy spam. Laplace smoothing (α=0.1) handles unseen words gracefully.

03 — Ensemble Voting

Both models independently classify every message. If they agree, the confidence score is the average of both. If they disagree, SVM wins as the tiebreaker (it typically performs better on text classification tasks), but the confidence is reduced to flag the uncertainty.
